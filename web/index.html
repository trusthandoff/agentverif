How it works

Three steps from source to certified. Fully automated, fully auditable.

1

Scan

Your agent package is automatically scanned against the OWASP LLM Top 10. A score below 70 is refused — no exceptions.

score ≥ 70 required
2

Sign

A unique License ID is generated and a cryptographic hash of your agent is registered on the agentverif.com registry. A SIGNATURE.json is injected.

AV-XXXX-XXXX
3

Certify

Buyers verify the certificate at verify.agentverif.com or via the CLI. Tampered agents are flagged instantly.

sha256 hash-locked

CLI quickstart

As a vendor

agentverif-sign sign ./agent.zip # Signed successfully # License: AV-84F2-91AB # Tier: indie

As a buyer

agentverif-sign verify ./agent.zip # VERIFIED # Verify: verify.agentverif.com/…

Output is JSON-compatible for CI/CD pipelines, LLM tool calls, and MCP integrations (agentverif-sign verify --json).

Why it matters

Why agentverif?

The AI agent ecosystem has no verification standard. That gap is a security liability.

Unsigned agents are a liability

Any agent without a verifiable provenance chain is a vector for supply chain compromise. Without certification, there is no way to distinguish safe from tampered code.

Buyers deserve to know what they're running

Enterprise buyers, security teams, and regulated industries cannot run unverified agents in production. Certification gives them the assurance they need to proceed.

One breach from an unverified agent = reputational damage

A single compromised agent incident can end a product's trust relationship with its users. Certification is not overhead — it is risk management.
